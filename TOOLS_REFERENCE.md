## Аналитика и отчетность

### 1. analytics_deal_pipeline
**Описание:** Получить воронку продаж - распределение сделок по стадиям

**Параметры:**
- `categoryId` (integer, опционально) - ID воронки (категории). По умолчанию 0
- `dateFrom` (string, опционально) - Дата начала периода (YYYY-MM-DD)
- `dateTo` (string, опционально) - Дата окончания периода (YYYY-MM-DD)

---

### 2. analytics_deals_at_risk
**Описание:** Получить сделки в риске (без активности более 3 дней или без изменений более 5 дней)

**Параметры:**
- `categoryId` (integer, опционально) - ID воронки
- `inactivityDays` (integer, опционально) - Дней без активности (по умолчанию 3)
- `noChangeDays` (integer, опционально) - Дней без изменения стадии (по умолчанию 5)

---

### 3. analytics_user_activity
**Описание:** Получить статистику активности пользователя (звонки, встречи, email, задачи, сделки, комментарии)

**Параметры:**
- `userId` (integer, **обязательно**) - ID пользователя
- `days` (integer, опционально) - Количество дней для анализа (по умолчанию 30)

---

### 4. analytics_team_performance
**Описание:** Получить сводную статистику по команде (все менеджеры)

**Параметры:**
- `days` (integer, опционально) - Количество дней для анализа (по умолчанию 30)
- `includeInactive` (boolean, опционально) - Включать ли неактивных пользователей

---

### 5. analytics_daily_summary
**Описание:** Получить ежедневный отчет: новые лиды/сделки, выполненные задачи, звонки

**Параметры:**
- `date` (string, опционально) - Дата для отчета (YYYY-MM-DD). По умолчанию сегодня

---

### 6. analytics_top_clients
**Описание:** Получить топ клиентов по сумме сделок

**Параметры:**
- `limit` (integer, опционально) - Количество клиентов в топе (по умолчанию 10)
- `dateFrom` (string, опционально) - Дата начала периода
- `dateTo` (string, опционально) - Дата окончания периода

---

### 7. analytics_conversion_rate
**Описание:** Рассчитать конверсию лидов в сделки

**Параметры:**
- `dateFrom` (string, опционально) - Дата начала периода
- `dateTo` (string, опционально) - Дата окончания периода

---

### 8. analytics_manager_low_activity
**Описание:** Получить список менеджеров со сниженной активностью

**Параметры:**
- `days` (integer, опционально) - Период для анализа (по умолчанию 7)
- `taskDecreasePercent` (integer, опционально) - Процент снижения задач (по умолчанию 30)
- `callDecreasePercent` (integer, опционально) - Процент снижения звонков (по умолчанию 25)

## CRM - Компании

### 22. crm_company_list
**Описание:** Получить список компаний с расширенной фильтрацией. Поддерживает фильтры по названию (TITLE), типу компании (COMPANY_TYPE: CUSTOMER, PARTNER, SUPPLIER, etc.), ответственному (ASSIGNED_BY_ID), индустрии (INDUSTRY), телефону (PHONE), датам создания/изменения. Операторы: '=', '!=', '>', '>=', '<', '<=', '%' (содержит), '@' (в массиве). Возвращает по 50 записей.

**Параметры:**
- `filter` (object, опционально) - Фильтр: TITLE, COMPANY_TYPE, ASSIGNED_BY_ID, INDUSTRY, PHONE, DATE_CREATE, OPENED (Y/N - доступна всем) и др.

- `order` (object, опционально) - Сортировка (например: {'DATE_CREATE': 'DESC', 'TITLE': 'ASC'})

## Примечания

- **Обязательные параметры** отмечены как "обязательно"
- **Опциональные параметры** можно не указывать
- Для **пагинации** используйте параметр `start` (смещение), обычно возвращается по 50 элементов
- **Фильтры** поддерживают операторы:
  - `=` - равно
  - `!=` - не равно
  - `>` - больше
  - `>=` - больше или равно
  - `<` - меньше
  - `<=` - меньше или равно
  - `%` - содержит (LIKE)
  - `!%` - не содержит
  - `@` - в массиве значений
- **Маски для select**:
  - `*` - все стандартные поля (без пользовательских)
  - `UF_*` - все пользовательские поля
